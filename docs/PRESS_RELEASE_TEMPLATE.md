# Генерация пресс-релизов

## Пользовательский сценарий

Редактор открывает раздел «Новости», загружает PDF/DOCX программу мероприятия, добавляет фото и короткие редакционные уточнения. Система извлекает текст и формирует JSON-черновик. Публикация возможна только после ручной проверки.

## Фактические ограничения

LLM не должна придумывать дату, место, должности, учёные степени, итоги и состав участников. Всё, чего нет в программе, попадает в `facts_to_verify`.

## Тон

Официальный академический стиль: без оценочных эпитетов, без рекламного «успешно/ярко/уникально», без преувеличения. Главная ценность — точность, ясная структура и уважение к научным статусам участников.

## Структура

Заголовок. Лид. Модератор/ключевые темы. Доклады. Дискуссия. Итоговый абзац. Подписи к фото. Список фактов для проверки.
